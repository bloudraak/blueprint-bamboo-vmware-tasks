Blueprint Bamboo VMwware Tasks

Contains tasks for Atlassian Bamboo to start, revert, shutdown and power off a virtual machine

Download this project as a .zip file Download this project as a tar.gz file

What is it?

Its a suite of tasks that allows you to automate virtual machines hosted by a VMware vSphere hypervisor

There are tasks for

  • Starting a virtual machine
  • Powering off a virtual machine
  • Shutting down the guest of the virtual machine
  • Reverting the virtual machine to a snapshot
  • Creating virtual machine snapshot

Scenarios

There are many times you'd like to run functional or integration tests after a build. However, deploying a build on the build server itself is generally a bad idea. These tasks allows you to define a virtual machine or three and deploy the build there. Since you can revert the virtual machine to a good snapshot, you can always revert the test machines back to a known state before deploying the build. It will reduce build failures.

Requirements

In order to run these tasks, you'll need

  • VMware vSphere 5.1 Unfortunately, the free edition of VMware vSphere hypervisor does not permit making changes to virtual machines. As a result, these tasks will not work against that version. You need a fully paid version of VMware vSphere for this to work. It may work with older versions, but it has not been tested.
  • Atlassian Bamboo 4.4 The tasks were tested with Atlassian Bamboo 4.4. It may work with previoous versions, but no guarentees are made.
